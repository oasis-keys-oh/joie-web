JOIE WEB PORTAL — Project Manifest
==================================

Project Location: /sessions/happy-youthful-pasteur/joie-web/

Complete Next.js 14 App Router web portal for Oukala Journeys traveler-facing trip viewer.

=== PROJECT STRUCTURE ===

app/
  layout.tsx              Root layout with header, footer, fonts (Inter + Playfair Display)
  page.tsx                Landing page (redirects, trip link instructions)
  globals.css             Global Tailwind CSS with custom colors
  not-found.tsx           404 page
  
  trip/[slug]/
    page.tsx              Trip overview page (hero, day cards, trip info)
    error.tsx             Error boundary for trip pages
    
    day/[day_number]/
      page.tsx            Day detail page (full itinerary, reservations, callouts)

components/
  TripHeader.tsx          Hero banner with trip title/subtitle
  DayCard.tsx             Clickable day card (region-colored, day number + title)
  DayHeader.tsx           Day header with number, date, region, location
  MealRow.tsx             Breakfast/lunch/dinner in horizontal row (color-coded)
  HotelCard.tsx           Hotel info card (name, address, check-in/out, confirmation)
  ReservationCard.tsx     Event/reservation card (title, time, address, status, briefing)
  NarrativeSection.tsx    Itinerary narrative segments (time label, headline, prose, logistics)
  CalloutBox.tsx          Styled callout boxes (WOW/Thread/One Thing/Phrase variants)
  DayNavigation.tsx       Previous/next day links + back to trip button
  PhotoSection.tsx        Photo with caption (Unsplash integration)

lib/
  types.ts                TypeScript interfaces for all Supabase entities
                          - Trip, TripDay, Event, ReferenceItem
                          - Nested JSONB types (ColorTheme, BriefingCard, etc.)
  supabase.ts             Supabase JS client + typed data fetching functions
                          - getTripBySlug()
                          - getTripDays()
                          - getTripDay()
                          - getDayEvents()
                          - getReferenceItems()
                          - getHotelForDay()
  colors.ts               Region color mapping utilities

Configuration Files:
  package.json            Dependencies (Next.js, React, Supabase, Tailwind, shadcn/ui)
  next.config.js          Next.js config (image optimization for Unsplash)
  tailwind.config.ts      Tailwind theme (navy, gold, region colors, fonts)
  tsconfig.json           TypeScript config (strict mode, path aliases)
  postcss.config.js       PostCSS with Tailwind/Autoprefixer
  .env.local.example      Environment template (user fills in real keys)
  .gitignore              Git ignore rules
  
Documentation:
  README.md               Complete setup, architecture, deployment guide
  PROJECT_MANIFEST.txt    This file

=== COMPONENTS BREAKDOWN ===

HERO & LAYOUT:
  TripHeader - Full-width image with overlaid trip title
  DayHeader - Day details with region highlight bar
  DayNavigation - Pagination between days, back to trip

DATA DISPLAY:
  DayCard - Grid/list item for day overview
  MealRow - 3-column meal display
  HotelCard - Hotel reference item with check-in/out
  ReservationCard - Event with status, confirmation, briefing
  NarrativeSection - Editorial itinerary content
  PhotoSection - Image with caption (Unsplash)

CALLOUTS:
  CalloutBox (variants: wow, thread, one_thing, phrase)
    - Each has distinct color scheme
    - Icon + label + title + content
    - Used for special moments/insights

=== PAGES ROUTING ===

Route: /
  Landing page with trip link instructions
  
Route: /trip/[slug]
  Trip overview
  Params: slug (web_slug from trips table)
  Shows: hero, dedication, trip dates, 15 day cards
  
Route: /trip/[slug]/day/[day_number]
  Day detail page
  Params: slug, day_number (1-15)
  Shows: day header, meals, hotel, events, narrative, callouts, navigation

=== SUPABASE INTEGRATION ===

Tables Queried:
  - trips (select by web_slug)
  - trip_days (select by trip_id and day_number)
  - events (select by day_id)
  - reference_items (select by trip_id, filtered by type)

All queries use error handling and single/multiple result expectations.
Nested JSONB fields parsed directly into TypeScript interfaces.

Auth: None (public URL access with secret web_slug)
Security: Supabase RLS policies recommended on production

=== ENVIRONMENT SETUP ===

Required environment variables:
  NEXT_PUBLIC_SUPABASE_URL=https://ticwkdkngtsklwlottzs.supabase.co
  NEXT_PUBLIC_SUPABASE_ANON_KEY=<user fills in>

Note: NEXT_PUBLIC_ prefix allows client-side access (safe with anon key)

=== STYLING SYSTEM ===

Colors (Tailwind custom theme):
  navy: #1B2B4B (primary, headings)
  gold: #C9A84C (accent, borders)
  region-specific: Dynamic (Andalusia orange, Barcelona purple, etc.)

Fonts:
  Inter (body text) - Google Fonts variable
  Playfair Display (headlines) - Google Fonts serif

Components:
  Meal colors: breakfast (yellow), lunch (blue), dinner (orange)
  Callout colors: wow (purple), thread (amber), one_thing (teal), phrase (rose)
  Status colors: confirmed (green), pending (yellow), cancelled (red)
